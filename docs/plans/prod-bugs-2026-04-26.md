## Bug A — Mongo index race trên `daily_routines`

### Stack trace

```
MongoDB.Driver.MongoCommandException: Command createIndexes failed:
  Index build failed: ... Collection InvestmentApp_prod.daily_routines
  :: caused by :: dropIndexes command.
```

Trace ID: `abaea8ec697036ca3de8aa6184d03812`, latency 1.013s, 2026-04-26T08:35:46Z.

### Root cause confirmed

[`src/InvestmentApp.Infrastructure/Repositories/DailyRoutineRepository.cs:11-27`](../../src/InvestmentApp.Infrastructure/Repositories/DailyRoutineRepository.cs#L11-L27):

```csharp
public DailyRoutineRepository(IMongoDatabase database)  // ← Scoped (per-request)
{
    _collection = database.GetCollection<DailyRoutine>("daily_routines");

    // Drop old unique index if it exists (legacy migration code, dead now)
    try { _collection.Indexes.DropOne("UserId_1_Date_1"); } catch { /* already gone */ }

    _collection.Indexes.CreateOne(...);  // race vs concurrent DropOne
    _collection.Indexes.CreateOne(...);
}
```

**Race scenario:**
1. Request 1 enters constructor → triggers `dropIndexes` on Mongo
2. Request 2 enters constructor concurrently → triggers `createIndexes`
3. Mongo rejects createIndexes vì có pending dropIndexes → `MongoCommandException`

`DropOne` này là **dead migration code** — đã chạy lúc đổi index từ `Unique=true` → `false`. Prod đã migrated từ lâu, không cần nữa.

### Anti-pattern lan rộng (out-of-scope hotfix)

25/25 repositories trong [`src/InvestmentApp.Infrastructure/Repositories/`](../../src/InvestmentApp.Infrastructure/Repositories/) đều gọi `Indexes.CreateOne(...)` trong constructor (Scoped). Bug A chỉ manifest ở DailyRoutine vì duy nhất nó có DropOne. Các repo khác idempotent → không crash, nhưng wasteful (DB roundtrip mỗi request).

### Fix — Hotfix (P0, branch `fix/daily-routines-index-race`)

Xoá legacy `DropOne` line. Race condition biến mất. ~5 phút fix.

```csharp
public DailyRoutineRepository(IMongoDatabase database)
{
    _collection = database.GetCollection<DailyRoutine>("daily_routines");

    var compoundIndex = ...;
    _collection.Indexes.CreateOne(new CreateIndexModel<DailyRoutine>(compoundIndex));

    var userIndex = ...;
    _collection.Indexes.CreateOne(new CreateIndexModel<DailyRoutine>(userIndex));
}
```

**Test:** mock `IMongoDatabase + IMongoCollection<DailyRoutine> + IMongoIndexManager<DailyRoutine>`, assert `DropOne` **không** được gọi (regression guard), và `CreateOne` gọi đúng 2 lần.

### Fix — Refactor (P1, sau khi hotfix merged)

Tạo `IndexInitializerService : IHostedService` chạy 1 lần lúc startup, ensure tất cả 25 collection's indexes. Repository constructor chỉ get collection, bỏ index management.

**Effort:** 4–6h.
**Benefit:** Hết wasteful per-request roundtrip, cleaner separation of concerns, dễ test.

## Bug B — `/api/v1/auth/google/callback` 500 (2×) ✅ FIXED

| # | Timestamp | Latency |
|---|---|---|
| 1 | 2026-04-26T08:35:44.979Z | 41 ms |
| 2 | 2026-04-26T12:54:28.664Z | 21 ms |

### Root cause

Bug report ban đầu nghi controller — **code review cho thấy `/api/v1/auth/google/callback` là `CallbackPath` của Google OAuth middleware**, không phải route controller. Controller route thật là `/api/v1/auth/google/process-callback` ([AuthController.cs:53](../../src/InvestmentApp.Api/Controllers/AuthController.cs#L53)).

→ Exception xảy ra **trong middleware** (correlation cookie missing, state mismatch, token-exchange rejection, …). Controller's try-catch không bao giờ chạy. Middleware-level errors bị nuốt vì **không có event handler** wired.

### Cookie SameSite check

Đã verify ở [Program.cs:295-296](../../src/InvestmentApp.Api/Program.cs#L295-L296): non-dev env dùng `SameSiteMode.None + CookieSecurePolicy.Always` — đúng cho cross-site (FE `onrender.com` ↔ API `run.app`). Không cần đụng.

### Fix applied

1. New `OAuthEventLogger` static helper ([src/InvestmentApp.Api/Auth/OAuthEventLogger.cs](../../src/InvestmentApp.Api/Auth/OAuthEventLogger.cs)) — structured log với path + correlationId + failure type/message, truncate message > 500 chars để chống log spam.
2. Wire `options.Events.OnRemoteFailure` trong `AddGoogle(...)` — gọi helper khi middleware fail.
3. Tests: 3 unit tests ([tests/InvestmentApp.Api.Tests/Auth/OAuthEventLoggerTests.cs](../../tests/InvestmentApp.Api.Tests/Auth/OAuthEventLoggerTests.cs)) — happy path, null exception, message truncation.

**Sau merge + deploy**, lần 500 tới sẽ có log chi tiết:
```
[ERR] Google OAuth remote failure at /api/v1/auth/google/callback (correlationId=abc-123, type=AuthenticationFailureException): Correlation cookie not found.
```
→ Đủ context để root-cause cụ thể (state vs cookie vs token).

## Bug C — Cold start 5–10s gây "FE treo" ✅ DOCUMENTED

`invest-mate-v2-bkk` config `--min-instances=0` → idle 15 phút → scale 0 → cold-start 5–8s.

### Fix options

| Option | Cost | Cold-start impact |
|---|---|---|
| Keep `Min: 0` | $0 | 5–10s mỗi 15 phút idle |
| `--min-instances=1` | ~$15–25/tháng | 0 cold-start |
| **Cron warmup ping** | $0.10/tháng (job thứ 4) | Cold-start chỉ ngoài giờ chợ |

### Recommended: cron warmup

```bash
gcloud scheduler jobs create http invest-mate-warmup \
  --location=asia-southeast1 \
  --schedule="*/14 1-8 * * 1-5" --time-zone="UTC" \
  --uri="https://invest-mate-v2-bkk-567557889111.asia-southeast3.run.app/health" \
  --http-method=GET
```

→ Mỗi 14 phút (08:00–15:59 ICT, T2–T6) GET health endpoint giữ ấm. Ngoài giờ accept cold-start.

⚠ Free tier Scheduler = 3 jobs hết. Job thứ 4 = ~$0.10/tháng — chấp nhận được.

**Defer:** không urgent. Nếu user không complain thêm, cân nhắc giữ Min:0 vì miễn phí.

### Action documented (this PR)

`worker-to-scheduler-deploy-guide.md` § Step 8 chứa gcloud command sẵn sàng chạy. Owner kích hoạt warmup khi nhận đủ phàn nàn cold-start. Dùng `/health/live` (không ping Mongo) thay vì `/health` để giữ ping rẻ.
